Открытое программное обеспечение

Если есть, что отправлять, то работу оформить в виде репозитория, положить в (zip, tar)-архив и отправить на электронный адрес преподавателя.

Архив именовать следующим образом: фамилия, тема, номер работы. Например, архив Chirkov_git4.zip должен содержать работу №4 по теме git студента с фамилией Чирков.

Письмо оформлять следующим образом:

  • тема письма: Короткое название группы. Фамилия. ПР №(номер практической работы).
  • тело письма: 
              Добрый день
          Номер группы. ФИО. ПР №(номер практической работы).
          [Ваш текст]
          Приложения:
            название файла - описание
            Например
            img01.png - результат проверки подписи коммитов

Невалидные письма могут остаться не замеченными.

Практическая работа №4. Git. Конфигурация

 --system (/etc/gitconfig)
--global (~/.gitconfig)
--local (.git/config)

git config --global user.name “Your Name”

Больше инофрмации man git config

Задания:

  1. установить имя пользователя GIT глобально;
  2. установить почтовый адрес для пользователя;
  3. установить различные текстовые редакторы по умолчанию (nano, vim) для разных проектов;
  4. настроить работу через прокси;
  5. добавить алиас для команды git log --oneline --all --graph --decorate.

Практическая работа №5. Git. Перемещение данных

 git rebase [ключи] - общий вид команды

Примеры:

git rebase testing
git rebase -i testing
git rebase -i HEAD~1
git rebase -i <commit>~1

Задания:

  1. Создать ветку iss01, а от нее — iss02.
  2. Переключиться на ветку iss01 и сделать несколько коммитов, тоже проделать с веткой iss02.
  3. Выполнить перемещение коммитов веток iss01 и iss02 в ветку master.

Практическая работа №6. Git. Метки (теги)

 git tag [ключи] [параметры] - общий вид команды

Легковесные (lightweight tag)
Снабженные комментарием (annotated tag)

Задания:

  1. придумать и добавить в ветке master 3 тега: 1 к последнему коммиту и 2 к коммитам, созданным ранее;
  2. изучить команду describe. Что она делает? Как ее можно использовать для использования тегов? (ответ поместить в файл readme внутри архива).

Практическая работа №7. Git. Подпись

Теория:

  • The GNU Privacy Guard (En)
  • Используем GPG для шифрования сообщений и файлов (Ru)
  • gpg + git (Ru)

Задания:

  1. создать gpg ключ (пара публичны/приватный);
  2. создать новый репозиторий и настроить его так, чтобы сгенерированный в п.1 ключ использовалася для подписи;
  3. создать 3 подписанных коммита;
  4. создать 1 подписанный тег;
  5. опубликовать репозиторий на сайте github.com;
  6. импортировать репозиторий одногруппника N (выполненный в рамках практической работы №7);
  7. импортировать публичный ключ одногруппника N;
  8. проверить подписи коммитов и тега.

В письме указать ссылку на свой репозиторий и репозиторий одногруппника N. К письму приложить свой публичный ключ и скриншоты результатов проверки подписи коммитов и тега.
